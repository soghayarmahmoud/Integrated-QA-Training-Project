# Contributing to Integrated QA Training Project

Thank you for your interest in improving this repository. Contributions help keep the project healthy, consistent, and useful for everyone.

## ✨ How to contribute

1. Review the available documentation:
   - `README.md`
   - `CODE_OF_CONDUCT.md`
   - `SECURITY.md`

2. Open an issue for:
   - bugs and test failures
   - improvements to test cases, reports, or documentation
   - new platform or workflow recommendations

3. Submit a pull request when you are ready.

## 🛠️ Contribution process

- Fork the repository and create a feature branch:
  - `git checkout -b feature/short-description`
- Keep commits small and focused.
- Write clear titles and descriptions for issues and PRs.
- Link related issues in your pull request description.
- Add or update documentation where relevant.

## 📋 Pull request checklist

- [ ] Code and documentation changes are clearly described.
- [ ] Changes are limited to a single concern.
- [ ] Formatting and structure are consistent.
- [ ] Any new details are reflected in the README or supporting docs.
- [ ] Existing functionality still works as expected.

## 🧹 Style guidelines

- Use simple, direct language.
- Keep documentation in markdown format.
- Use consistent section headings and lists.
- Prefer active voice for instructions.

## 🧩 Repository structure

Use the following structure for new content where appropriate:

- `docs/` — test plans, strategy, and process documentation
- `test-cases/` — structured test cases for UI, API, and DB work
- `test-reports/` — report templates and summary artifacts
- `scripts/` — automation scripts, SQL queries, and Postman collections
- `assets/` — supporting files, screenshots, and reference data

## 🤝 Community expectations

Follow the `CODE_OF_CONDUCT.md` and be respectful, constructive, and inclusive.

## 📞 Need help?

If you are unsure how to proceed, open an issue and we will help you get started.
