# Security Policy

## Supported Versions

This repository is actively maintained. Security issues should be reported for the current main branch.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue.

1. Create a private issue if possible.
2. If a private issue is not available, notify the repository maintainer directly.
3. Provide enough detail to reproduce the issue safely.

## What to Include

- A clear description of the issue.
- Steps to reproduce the problem.
- Impact and affected components.
- Suggested mitigation or workaround, if known.

## Response Process

Maintainers will evaluate reports promptly and respond with next steps.

## Responsible Disclosure

Please allow maintainers time to investigate and remediate before disclosing publicly.
